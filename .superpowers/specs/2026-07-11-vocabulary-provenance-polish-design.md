# FLQ-6.2 — Vocabulary: провенанс записей + LingQ-полиш: design spec

- Дата: 2026-07-11
- Задача: FLQ-6.2 (инкремент поверх FLQ-6 Increment 1, та же ветка
  `feature/FLQ-6-vocabulary-page`)
- Референсы: `.superpowers/specs/2026-07-08-vocabulary-page-design.md`,
  ADR-0005, Figma-макет «Vocabulary — Desktop 1440» (CSS-выгрузка от
  пользователя, 2026-07-11), `docs/ui/vocabulary.md`
- Решения пользователя: провенанс через поле происхождения (не фильтр по
  статусу); работаем в той же ветке; скоуп UI = полиш таблицы + sort-контрол;
  лемма/грамм-чипы, Импорт (FLQ-14), Повтор (FLQ-7), бейдж уровня у термина,
  топбар-чипы (стрик/монеты/AI), «Плейлисты» — НЕ входят.

## 1. Провенанс: `token_items.added_by` (ядро)

### 1.1 Модель и миграция

- Новое поле `added_by: Mapped[str]` (`String(16)`, NOT NULL,
  server_default `'user'`); значения `'user' | 'bulk'`.
- Миграция `0009_item_provenance`:
  1. `ADD COLUMN added_by VARCHAR(16) NOT NULL DEFAULT 'user'`.
  2. Бэкфилл: `'bulk'` для всех item id из
     `bulk_actions.payload_json->'token_item_ids'` (ключ пишет
     `reader_state/bulk.py:76`) у **не-undone** действий
     (`undone_at IS NULL`). Уже удалённые undo строки не существуют — noop.
  3. Индекс не нужен (фильтр всегда идёт вместе с user_id+language_code,
     существующий `ix_token_items_user_lang` покрывает).
- CHECK-констрейнт: `added_by IN ('user','bulk')`
  (`ck_token_items_added_by`).

### 1.2 Запись

- **`'bulk'` пишет только bulk-known ридера** (reader_state, кнопка
  «Далее»): создаваемые известными token_items получают `added_by='bulk'`.
  Undo как раньше удаляет созданные строки.
- **Все явные пути пишут `'user'`** и **промоутят существующие
  `'bulk'`-строки в `'user'`** (безвозвратно):
  - `vocabulary.create_item` (в т.ч. авто-создание по правилу FLQ-5 §2.1 и
    idempotent-ветка «existing»: при явном create поверх bulk-строки —
    промоушен);
  - `vocabulary.patch_item` (инлайн-пикер словаря, футер карточки);
  - `vocabulary.bulk_action` `set_known`/`set_ignored`/`add_tag` (действия
    со страницы словаря = явные);
  - добавление перевода/заметки/тега (item уже существует к этому моменту —
    промоушен делается в `_owned_item`? Нет: промоушен явный, в
    `add_translation`/`put_note`/`add_tag`/`update_translation`/
    `delete_translation` после `_owned_item` — одна общая приватная
    функция `_promote_to_user(item)`).
- Bulk-known поверх существующей `'user'`-строки НЕ понижает провенанс
  (существующие строки bulk-known вообще не трогает — как сейчас, он
  «only new»).

### 1.3 Чтение

- `GET /api/vocabulary` получает параметр `added_by: 'user' | 'all'`,
  **default `'user'`**. `'user'` добавляет `WHERE added_by = 'user'`;
  `'all'` — без фильтра.
- `LookupResult`/reader token-statuses/подсветка ридера — без изменений
  (bulk-known слова остаются «изученными» в текстах).
- Frontend: `VocabListParams.added_by?: 'user' | 'all'`; страница шлёт
  значение из store (`showAuto: boolean`, default false → `'user'`).
- FilterPopover: чекбокс «Показывать авто-изученные» → `showAuto`
  (сбрасывает page/selection как остальные фильтры; `resetFilters`
  возвращает false; `filtersAreDefault` учитывает его).

## 2. Визуальный редизайн (Figma-токены)

### 2.1 Токены

CSS-переменные в `frontend/src/index.css` (блок `/* vocabulary (FLQ-6.2) */`;
светлая тема, тёмная — вне скоупа):

```css
--vocab-page-bg: #F5F7FA;
--vocab-card-border: #E0E6EB;
--vocab-term-fg: #1A2133;
--vocab-muted-fg: #6B7585;
--vocab-header-fg: #8C96A3;
--vocab-translation-fg: #4580F2;
--vocab-chip-gram-bg: #EDF0F5;   /* грамм-чипы (задел, в этом инкременте не используются) */
--vocab-chip-pos-bg: #F5EDDB;    /* POS-чипы */
--vocab-picker-border: #C7CCD4;
--vocab-picker-active-bg: #FFDB8C;
--vocab-picker-active-fg: #523812;
--vocab-known-bg: #45A86B;
--vocab-subtabs-track: #E6EBF0;
```

Шрифт: font-stack `'Inter', system-ui, sans-serif` на странице словаря
(без добавления пакета шрифта; у кого Inter установлен — получает макетный
вид, фолбэк системный). Отдельная зависимость @fontsource — вне скоупа.

### 2.2 Строки-карточки вместо таблицы

- Desktop-список = вертикальный стек **карточек** (одна на слово): белый
  фон, бордер `--vocab-card-border`, radius 8px, зазор 8px, паддинги по
  макету (высота контентом: 1 строка контекста ≈ 88px, 2 строки ≈ 110px).
- Внутренняя сетка колонок сохраняется (grid): ☐ / термин+чипы / перевод /
  исходный текст / пикер — ширины ориентировочно по макету (термин ~300px,
  перевод ~260px, текст ~330px, пикер ~250px, гибко через fr/minmax).
- Заголовок-строка над карточками: чекбокс select-all + подписи колонок
  **капсом**: `ТЕРМИН / ПЕРЕВОД / ИСХОДНЫЙ ТЕКСТ / СТАТУС` — 11px,
  weight 600, letter-spacing 0.5px, цвет `--vocab-header-fg`. В этой же
  строке справа: «Показать: 25 ▾» и нумерованная пагинация (см. §3.1).
- Термин: 15px semibold `--vocab-term-fg`; под ним чипы: POS
  (`--vocab-chip-pos-bg`) + пользовательские теги (нейтральный
  `--vocab-chip-gram-bg`), 11px, radius 4px, высота 20px.
- Перевод: флажок-эмодзи target-языка (🇷🇺) + текст 14px
  `--vocab-translation-fg`; нет перевода — «—» серым.
- Исходный текст: 13px `--vocab-muted-fg`, в кавычках, максимум 2 строки
  (line-clamp-2), обрезка ~80 симв. остаётся.
- `aria`-роли: сохраняем семантику списка (role="table" не обязателен;
  допустимо перейти на role="list"/"listitem" — существующие тесты таблицы
  обновляются соответственно, поведенческие колбэки не меняются).
- Mobile `VocabularyCardList` перенимает те же токены (карточки уже есть).

### 2.3 ConfidencePicker (общий рестайл)

Общий компонент — новый вид ВЕЗДЕ (словарь, WordCard-футер, mobile):

- Круги: бордер `--vocab-picker-border`; размеры **md 32px, sm 28px**
  (в макете пилюли 28px, кнопка ✓ 32px — берём 28 как sm для строк
  словаря, 32 как md для WordCard).
- Активный уровень 1–4: заливка `--vocab-picker-active-bg`, текст
  `--vocab-picker-active-fg`, без бордера.
- `✓` активный (known): заливка `--vocab-known-bg`, белая галочка.
- `🗑` активный (ignored): как сейчас — тёмная обводка (в макете нет
  активного состояния корзины).
- aria-label'ы и поведение НЕ меняются (существующие тесты по ролям
  проходят; классовые ассерты в тестах компонента обновить под новые
  классы).

### 2.4 Сабтабы и тулбар

- Табы страницы — **segmented control**: трек `--vocab-subtabs-track`
  radius 8px, активный сегмент — белый, бордер `#C7CCD4`, лёгкая тень,
  radius 6px; состав табов прежний (Все / Слова / Фразы / К повторению,
  последние два disabled).
- Тулбар в одну строку с сабтабами, справа: «⌕ Поиск» (клик раскрывает
  инпут; на desktop ≥lg инпут может быть виден сразу — реализация вправе
  выбрать постоянный инпут), «⊟ Фильтры», «Ещё действия ▾» (переименование
  bulk-меню; с count: «Ещё действия (2)», disabled при 0). Текстовые
  кнопки 13px без рамок.

## 3. Функциональный полиш

### 3.1 Нумерованная пагинация

- Кружки-номера страниц: активная — тёмный круг (`--vocab-term-fg`) с белой
  цифрой (20px), остальные — серые цифры; окно: 1 … (p-1) p (p+1) … N с
  многоточиями; ‹/› остаются по краям.
- Расположение: правый край строки заголовков (по макету) + дублирование
  внизу списка (удобство; вправе упростить до только-верхней, если тесно).
- Кламп: если после мутаций/фильтров `page > totalPages` — автоматически
  `setPage(totalPages)` (закрывает follow-up финального ревью).
- «Показать: 25 ▾» — подпись селектора page-size.

### 3.2 Sort-контрол

- Дропдаун в тулбаре: «Сначала новые» (default, created_at desc) /
  «Сначала старые» (asc) / «А–Я» (text asc) / «Я–А» (text desc) →
  `store.setSort/setSortDir`; смена сортировки сбрасывает page на 1
  (добавить в сеттеры store).

## 4. Тестирование

- **Backend:** миграция-бэкфилл (создать items + bulk_action c payload →
  after upgrade помечены 'bulk', прочие 'user'); bulk-known создаёт
  'bulk'; create_item/автосоздание — 'user'; промоушен bulk→user на
  patch_item, add_translation, add_tag, bulk_action set_known;
  list_items фильтр added_by (default user, all возвращает оба);
  bulk-known не трогает существующую user-строку.
- **Frontend:** нумерованная пагинация (окно, эллипсисы, активная, кламп);
  sort-дропдаун меняет параметры запроса и сбрасывает page; чекбокс
  «Показывать авто-изученные» шлёт added_by=all; переименования
  (ИСХОДНЫЙ ТЕКСТ, Ещё действия, Показать:); рестайл-смоук ConfidencePicker
  (активные классы). Существующие тесты обновляются под новые
  label'ы/классы, поведенческие ассерты сохраняются.
- **Смоук (живой браузер):** пролистать страницу урока «Далее» → слова НЕ
  появляются в словаре; включить «Показывать авто-изученные» → появились;
  поставить слову уровень пикером в ридере → слово появилось в словаре без
  чекбокса; визуальная сверка карточек/пикера/сабтабов с макетом.

## 5. Не входит

Лемма/грамм-чипы (словоформы Kaikki — отдельная задача при FLQ-15),
«Импортировать словарь» (FLQ-14), «Повтор лексики» (FLQ-7), топбар-чипы
геймификации, «Плейлисты», бейдж уровня у термина, тёмная тема для новых
токенов, пакет шрифта Inter.
