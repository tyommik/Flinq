# Reader (per sentence) — доработка дизайна под макет Desktop 1440

Дата: 2026-07-12
Статус: approved

## Цель

Привести страницу урока (режим «по предложениям») к Figma-макету «Reader (per sentence) — Desktop 1440». Логика ридера (хоткеи, синхронизация позиции, undo, запросы) не меняется — только представление и мелкие композиционные изменения.

## Объём

Всё из макета, **кроме** геймификации в TopBar (чипы стрика/монет/AI-токенов/аватар с бейджем — под них нет бэкенда; существующий `AppTopBar` не трогаем). Page-режим (`PageView`) не трогаем, только проверяем, что общие тулбары не разъехались.

Аудио-элементы макета (Play над текстом, 🔊 у слов, «Сгенерировать аудио») — **заглушки**: рисуем по макету, disabled, тултип «Скоро».

## Дизайн

### 1. SubToolbar (`ReaderTopBar.tsx`)

Строка высотой 56px:

- Слева — крестик «×» (выход в библиотеку), как сейчас.
- По центру — прогресс-трек шириной до 700px, высота 4px, скруглённый, фон `#E8EBED`; зелёный филл (`primary`) с точкой-маркером 10px на конце филла.
- Справа — «Aa» (существующий дропдаун настроек шрифта) и переключатель боковой панели (только в page-режиме, как сейчас).
- Кнопку «⋯» из макета **не рисуем** до появления функций.
- **Название урока из тулбара убираем** — в макете его нет.

### 2. Текст предложения (`SentenceView.tsx`)

- Центрированная колонка ~900px; текст выровнен **по левому краю** (сейчас — по центру).
- Шрифт 20px, line-height 1.8 (36px), текущий санс (Inter).
- Над текстом — круглая кнопка Play 54px с бордером `1.5px #D9DBE0`, по центру колонки; disabled-заглушка.
- Инлайновые стрелки ‹ › рядом с текстом убираем (навигация — секция 6).

### 3. Подсветки слов (`TokenSpan.tsx`, `globals.css`)

- Скругление 4px, горизонтальный паддинг 4px с компенсирующим отрицательным маргином (не рвать ритм пробелов).
- Цвета из макета: изучаемые (tracked) `#FFEBA8`, новые `#C6DFFF`. Обновить значения CSS-токенов `--reader-tracked*` / `--reader-new*` в `globals.css` (имена токенов не меняются).

### 4. «Показать перевод» и перевод (`SentenceView.tsx`)

- Ссылка 14px, подчёркнутая, цвет `muted-foreground`, шеврон ▾; выровнена по левому краю текстовой колонки.
- Раскрытый перевод — курсив 14px, по левому краю. Бейдж «AI» и обработка ошибок/повтора — как сейчас.
- Логика запроса (`useSegmentTranslation`, ленивая загрузка по клику) не меняется.

### 5. Список лексики (`SentenceVocabList` — новый компонент)

Вертикальный список строк вместо чипов. Показываем слова предложения со статусом **tracked и new** (known/ignored — нет). Дедупликация по нормализованной форме, порядок появления в предложении.

Строка:

- Кружок статуса 28px: жёлтый `#FFDB8C` с цифрой уверенности (13px semibold, цвет `#523812`) для tracked; синий `#C6DFFF` с точкой для new.
- Слово 16px medium + иконка 🔊 (disabled-заглушка).
- Под словом — перевод 14px `muted-foreground`; если перевода нет — строка без подзаголовка.
- Между строками разделитель 1px `#E8EBED` с отступом слева от текстовой части.

Переводы: `useQueries` → `vocabularyApi.lookup(lang, text, target)` по слову (кэш react-query, слов в предложении немного). Показываем `translations.primary?.text`.

Клик по строке открывает WordCard (как раньше по чипу).

### 6. Нижняя панель (`BottomToolbar.tsx`)

Высота 80px, верхний разделитель. Три действия по сетке слева/центр/справа, каждое — иконка над подписью 14px:

- «♪ Сгенерировать аудио» — disabled, тултип «Скоро».
- «📖 Показать всю страницу» ↔ «По предложениям» — переключение режима (работает, как сейчас).
- «✓ Повторить лексику» — disabled, тултип «Скоро (FLQ-7)».

### 7. Навигация (`ReaderPage.tsx` / `SentenceView.tsx`)

- Большие стрелки ‹ › (28px, `muted-foreground`) фиксированно по краям экрана, вертикально по центру вьюпорта — для sentence-режима.
- Disabled-состояние на границах (первое/последнее предложение).
- Хоткеи и свайпы не меняются. Page-режим и стрелки `PageView` — без изменений.

## Тесты

- Реальное аудио/TTS.
