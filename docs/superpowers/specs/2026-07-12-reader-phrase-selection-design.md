# Выделение фраз в ридере — дизайн

Дата: 2026-07-12
Статус: утверждён (brainstorming с пользователем)
Связанные документы: ADR-0001 (unit of learning, Phrase — first-class сущность),
ADR-0005 (модель статусов), docs/lingq-like-self-hosted-spec-2026.md §фразы.

## Цель

Пользователь выделяет мышью последовательность слов в ридере и получает
карточку фразы: AI-перевод в контексте, свой перевод, статус, заметки, теги —
как у слова. Сохранённые фразы подсвечиваются во всех текстах, где встречается
та же последовательность слов, и открываются по клику. Слова внутри фразы
остаются самостоятельными кликабельными единицами.

## Ключевые решения

| Вопрос | Решение |
| --- | --- |
| Объём фичи | Полный цикл: выделение → перевод → сохранение → подсветка → клик |
| Границы фразы | Только внутри одного предложения (обрезается предложением якоря) |
| Лимит длины | 2–8 **слов**; пунктуация входит в поверхностный текст, но не в лимит |
| «don't», «well-known» | Один токен (ADR-0001), выделение не рвёт их |
| Устройства | Только мышь в этой итерации; тач — отдельной задачей |
| Матчинг подсветки | Везде по совпадению нормализованной последовательности слов (все уроки) |
| Матчинг и пунктуация | Пунктуация прозрачна: «so far, so good» ≡ «so far so good» |
| Пересечения вхождений | Не поддерживаются в v1: leftmost-longest выигрывает |
| Карточка | Та же док-панель, что у слов; Wiktionary-секции у фраз нет, AI — есть |
| Архитектура матчинга | Вариант A: матчинг на клиенте; серверные occurrences — когда дойдём до SRS фраз |

## UX

### Hover

Любое слово при наведении получает обводку по периметру (ring/outline) поверх
существующей статусной подсветки — в том числе слова внутри фразы.

### Drag-выделение (мышь, ЛКМ)

1. `pointerdown` на слове запоминает якорный ординал. Клик без движения до
   другого слова работает как сейчас — открывает карточку слова.
2. Дотянул до другого слова → режим выделения: диапазон от якоря до текущего
   слова, округлённый до целых слов, подсвечивается временным фоном.
   Направление любое (справа налево нормализуется). На время драга —
   `user-select: none`.
3. Диапазон обрезается границами предложения якоря и клампится на 8 словах:
   тянуть дальше можно, выделение не растёт.
4. `pointerup`: если в диапазоне ≥2 слов — открывается карточка фразы.
   Esc во время драга отменяет выделение.

### Сохранённые фразы в тексте

- Подложка фоном под группой слов, цвет по статусу (как у слов).
- Слова внутри самостоятельны: hover — обводка слова, клик — карточка слова.
- При наведении на зону фразы подложка расширяется геометрически (выступает за
  периметр слов, цвет не меняется); клик по расширенной кромке — карточка
  фразы. Расширение не сдвигает геометрию текста (padding + отрицательные
  margin).
- Пересекающиеся вхождения не подсвечиваются: при матчинге действует
  leftmost-longest, вложенность/пересечение пропускается. Сохранить
  пересекающуюся фразу можно — она просто не подсветится там, где конфликтует.

## Модель данных и бэкенд

### Таблица `phrase_items` (новая, одна alembic-ревизия)

Зеркало `token_items`:

- `id`, `user_id`, `language_code`, `status` (tracked/known/ignored),
  `confidence`, `added_by`, `created_at`, `updated_at`; constraints как у
  `token_items` (confidence 0–5, `tracked ⇔ confidence NOT NULL`).
- `phrase_text` — нормализованный join-key: нормализованные формы
  слово-токенов, соединённые одним пробелом (`so far so good`). Пунктуация и
  оригинальные пробелы не входят.
- `display_text` — поверхностный текст выделения с пунктуацией
  («so far, so good») для карточки и страницы Vocabulary.
- `UNIQUE (user_id, language_code, phrase_text)`.
- CHECK: 2–8 слов (по числу пробелов в `phrase_text`).

Сателлиты (`personal_translations`, `personal_notes`, `item_tags`) уже
полиморфные по `item_kind` — не меняются.

### Генерализация vocabulary service

- `_check_kind` принимает `phrase`; `item_kind` прокидывается параметром во
  все операции (create/patch/translations/notes/tags) вместо захардкоженного
  `"token"`.
- `create_item` при `kind=phrase` принимает сырое выделение, нормализует в
  `phrase_text`, сохраняет `display_text`.
- API-контракт (`/api/vocabulary/items/{kind}/{id}/...`) не меняется —
  меняется только реализация.

### Lookup и список фраз

- `GET /api/vocabulary/lookup` получает параметр `kind` (default `token`).
  При `kind=phrase`: поиск `phrase_items` по нормализованному тексту, ответ в
  той же форме (item_id, status, confidence, translations, note, tags),
  словарной (Wiktionary) части нет.
- Новый endpoint `GET /api/vocabulary/phrases?lang=` →
  `[{item_id, phrase_text, status, confidence}]` — лёгкий список для матчинга
  в ридере.

### Прочее

- AI-перевод: без изменений — `aiApi.translate` уже принимает `surface_text`
  (display-текст фразы) + контекст предложения.
- Vocabulary-страница: фразы попадают в общий список (`kind: 'all'`),
  `VocabListItem.kind` расширяется значением `'phrase'`, bulk-операции
  работают по `item_kind`.

## Архитектура фронтенда

### Новые модули (`features/reader/`)

- **`phraseMatching.ts`** — чистая логика: индекс
  `Map<первое слово, фразы[] по убыванию длины>`;
  `matchPhrases(tokens, index)` → непересекающиеся вхождения
  `{fromOrdinal, toOrdinal, phrase}` по leftmost-longest. Матчинг только по
  слово-токенам (`n`), пунктуация/пробелы прозрачны. Мемоизация на
  предложение.
- **`usePhraseSelection.ts`** — автомат драга:
  `idle → armed (pointerdown) → dragging (pointerover другого слова) → done
  (pointerup)`. Держит `anchorOrdinal`, `currentRange` (кламп предложением и
  8 словами), отдаёт `dragRange` для подсветки и колбэк с готовым выделением.
  Esc — отмена. Только `pointerType === 'mouse'`. Обработчики — делегированием
  на контейнере предложения.
- **`PhraseSpan.tsx`** — обёртка вхождения: инлайновый спан с подложкой по
  статусу и расширяющейся hover-зоной. Клик по обёртке — карточка фразы;
  клики по словам внутри — `stopPropagation`.

### Изменения существующего

- **`TokenSpan`** — hover-обводка, класс временного выделения при драге.
- **Рендер предложения** (общий для `PageView`/`SentenceView`) — вместо
  плоского `tokens.map(TokenSpan)` рендер «ранов»: обычные токены + фразовые
  группы из `matchPhrases`.
- **`ReaderPage`** — `selectedWord` становится
  `selectedItem: { kind: 'token'|'phrase', text, displayText, sentenceText,
  ... } | null`. Для фразы `text` — нормализованный join-key, `displayText` —
  срез поверхностных токенов.
- **`WordCard` + `useWordCard`** — принимают `kind`: lookup и мутации
  прокидывают его в API; Wiktionary-секция только для `kind=token`,
  AI-подсказки — для обоих.
- **`api/vocabulary.ts`** — `lookup(..., kind)`, `phrases(lang)`,
  `createItem` с `kind: 'token'|'phrase'`.

### Поток данных

Ридер грузит content + token-statuses + `phrases(lang)` (react-query, ключ
`['phrases', lang]`) → мемо-матчинг по предложениям → рендер. Сохранение фразы
в карточке → инвалидация `['phrases', lang]` и lookup → подложка появляется
во всех вхождениях сразу.

## Обработка ошибок

- **AI недоступен (503/ошибка)** — деградация как у слов: «AI-переводы
  отключены» / «Повторить»; карточка остаётся рабочей.
- **Дубль фразы** — карточка всегда делает lookup по нормализованному тексту
  перед созданием, существующая фраза открывается со своим `item_id`. Гонка →
  409 → фронт перезапрашивает lookup.
- **Валидация на бэкенде** — 2–8 слов, непустой текст после нормализации
  (границы предложения бэкенд не проверяет — он про предложения не знает).
  Ошибка → существующий тост «Не удалось сохранить».
- **`phrases(lang)` не загрузился** — ридер работает без подсветки фраз,
  чтение не блокируется, query ретраится по умолчанию.

## Тестирование

- **Backend (pytest):** миграция и constraints (уникальность, 2–8 слов,
  confidence); сервис с `kind=phrase` (create/patch/lookup/translations/
  notes/tags); нормализация `phrase_text` (пунктуация выпадает, `don't` — одно
  слово); `GET /api/vocabulary/phrases`; vocabulary list с фразами.
- **Frontend (vitest):** `phraseMatching` — leftmost-longest, пересечения,
  пунктуация внутри, фраза с `don't`, границы предложения;
  `usePhraseSelection` — клик ≠ драг, кламп 8, обратное направление, Esc;
  `WordCard` с `kind=phrase` — без словарной секции, мутации с правильным
  kind; рендер ранов — группа фразы + клик по слову внутри открывает слово.
- **Ручная проверка (playwright-cli):** драг мышью на реальном уроке,
  hover-зоны фразы, сохранение → подложка во всех вхождениях.

## Вне объёма (v1)

- Тач-устройства (long-press выделение) — отдельная задача.
- Серверные phrase occurrences и статистика — когда дойдём до SRS фраз.
- Подсветка пересекающихся/вложенных вхождений.
- Фразы через границу предложения.
